Overview

TYPO.py is a simple program generating source files which can be used in your own projects.

Source files are created from templates. Templates can contain placeholders in a form of ${placeholder_name}. When file is generated, placeholders are filled by program written in Python. Python code can read and interpret values set by TYPO interpreter.

Typo is:

  • Easy to use,
  • Extendable,
  • Keep user code

Easy to use

To use TYPO, you need to create template containing set of placeholders. Tou can take any of your typical file, and change some of its content to placeholders. Then you need to run TYPO and generate subsequent files by setting placeholder values and running generation by typing template name. So you have two types of orders:
  • assignment - order is in the form name=value, where name is any string identifier and value can contain placeholders which will be evaluated at the moment of using this value.
  • generator - order is in the form generator, and runs generation of the output file based on template named generator.template. When generation is started, some values need to be defined:
    • path - path where the output file will be located
    • file_name - full name of the file
    • template_path - path to the folder with templates

You can also store list of TYPO commands in the file and use it as a parameter when starting TYPO.

Placeholders can be filled by the assigned values, but more powerfull is runnging python code instead. When placeholder is found in the template, TYPO checks if Python class named gen_placeholder_name. If it exist, it is created and its method generate is called. TYPO passes to the function repository of all assigned values and the output object allowing writing into the file. This writing object also keeps proper indentation og generated code.

Extendable

Programmer using TYPO can crate not only its own templates, but also your own code generators. It is even possible to build your own sowtware which will use TYPO as its output. More detais you can find in TYPO user reference manual

You can not only run TYPO as standalone program, but also as a library for your metaprograms. Just take alook at TYPO programmer reference manual.

Keep user code

It is not recommended to mix generated code with code written by the programmer, however sometimes it might be handy.

User code can be placed in generated files in places specified by the template. When creating templates, you can locate placeholder ${user_code} in any place of the template. It will be replaced by the code read from the modified code. User code placeholder muse be the only one element in the line of template file. Its indentation have no result in copied code, which is moved 'as it is'.

User code placeholder must be surrounded by constant lines easy to distinguish from other lines. Usually - such lines should be comments informing about user code placement possibility.
